Vercel
Deploy y hosting — frontend + backend
publi vive 100% en Vercel. El frontend (React, Next.js) y el backend (API Routes en app/api/) se deployean juntos desde el mismo repo de GitHub. Cada push a main dispara un deploy automático. Los PRs generan URLs de preview. No hay servidor separado — todo son Serverless Functions con timeout de 10 segundos en el tier gratuito.
Serverless Functions Auto-deploy desde GitHub Preview por PR 10s timeout (tier free)
Visión general 02
Cómo se conecta todo

El usuario interactúa con Next.js en Vercel. El backend llama a Supabase para datos, Vercel Blob para archivos, QStash para scheduling, Groq para IA, e Instagram para publicar.

Row Level Security: todas las tablas tienen RLS habilitado con la política auth.uid() = user_id. Aunque un endpoint tenga un bug, Supabase garantiza que los datos de un usuario nunca sean visibles para otro.
